# AGENTS

本文件约定本项目中 AI 助手和开发者的协作规则。

## 提交与推送规范

- 提交信息、推送说明、变更摘要默认使用中文。
- Git commit 使用 Conventional Commits 风格：`<type>(可选范围): 中文摘要`。
- `type` 保持英文小写，摘要使用简洁中文，不使用句号结尾。
- 一次提交只表达一个清晰意图，避免把无关改动混在一起。
- 推送前先确认 `git status --short` 没有遗漏文件。

## 常用 type

- `feat`: 新功能或新增用户可感知能力。
- `fix`: 修复缺陷、异常行为或线上问题。
- `docs`: 文档、说明、注释类改动。
- `style`: 仅格式、排版、空格等不影响逻辑的改动。
- `refactor`: 重构代码但不改变外部行为。
- `test`: 新增或调整测试。
- `chore`: 构建、依赖、脚手架、仓库维护等杂项。
- `ci`: CI/CD 流程与配置改动。
- `build`: 构建系统、打包配置或外部依赖改动。
- `perf`: 性能优化。
- `revert`: 回滚已有提交。

## 示例

```text
feat(frontend): 新增个人技术记录首页
fix(backend): 停用公开联系提交接口
docs: 补充个人备案站部署说明
test: 更新站点内容接口断言
chore: 调整项目协作规范
```

## 验证习惯

- 代码改动后按影响范围运行测试或构建。
- 文档-only 改动可不跑完整测试，但需要检查格式和链接。
- 推送完成后确认远端分支已经更新。
